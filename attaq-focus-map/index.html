A visual workspace for noisy projects

Why this app exists

Projects often feel bigger than they really are. A big goal turns into a foggy cloud of half-remembered tasks, unfinished branches, blocked chunks, and things you promised to return to later.

This app turns that cloud into a living map. You do not write long task descriptions. You create small connected nodes, split big work into visible pieces, focus on one branch when the full map is too much, and close nodes when a piece is done.

What it is good for

Breaking projects into subprojects, seeing what belongs where, moving branches when plans change, and keeping progress visible without building a heavy task database.

What it avoids

No required deadlines, no giant forms, no assignees, no status spreadsheets, no fake productivity ritual. A node is just a piece of the project.

How it works

  • Root nodes are projects. Create them with + Project.
  • Child nodes are smaller pieces. Right-click a node and use + Child node.
  • Double-click a node to focus on it. Everything above it is hidden; its children stay visible.
  • Group focus lets you select several nodes from different branches and focus only on them and their children.
  • Close nodes when a piece is done. Closed nodes stay on the map, so progress is visible.
  • Defer a branch to dim it without deleting it. Returning it restores the original colors.
  • Move a branch when a chunk of work belongs somewhere else now.
  • Autosave writes every action to the active JSON map file in the MAPS folder.
